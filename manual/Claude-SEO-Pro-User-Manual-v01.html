1. What Claude SEO Pro is

Claude SEO Pro is an SEO analysis system that runs inside Claude Code (the terminal AI assistant). You type plain commands like /seo audit https://yourclient.com and it produces a prioritised, evidence-backed SEO report — the kind of work that normally takes a senior consultant hours, in minutes.

It is built for one SEO manager to do the work of a team: full technical audits, content & E-E-A-T review, schema, Core Web Vitals, AI-search readiness, keyword research, local/GBP, and competitor analysis — all from one place, with the data living on your own machine.

What makes the "Pro" version different

  • Guided onboarding — a wizard walks you through connecting your data APIs securely (covered in sections 4–6).
  • A smarter full-audit pipeline — it understands the business first, then audits, then researches keywords, then checks local presence (section 8).
  • An anti-fabrication guarantee — it will never invent a number. If it can't verify something, it tells you (section 2).
You don't need to be technical. If you can type a command and read a report, you can run this. This manual assumes no coding knowledge. Where a command is shown, you copy it exactly.

2. How it thinks: skills, agents & the no-fabrication rule

Three ideas explain everything the system does. Understanding them takes two minutes and makes the rest obvious.

Skills = the commands you type

A "skill" is a capability you trigger with a slash command. /seo audit, /seo schema, /seo-setup are all skills. You'll see the full list in section 7.

Agents = the specialists it sends out

When you run a big job like a full audit, the system dispatches several specialist agents in parallel — one for technical SEO, one for content, one for schema, and so on. Each examines the site and reports back. You don't manage them; they work behind the scenes. It's why a full audit is both fast and thorough.

The no-fabrication rule (the "Evidence Integrity Protocol")

This is the most important thing to trust. The system is built so that every finding traces back to real evidence — a line in the site's code, or a field in a real API response. If a page can't be fetched, or an API isn't connected, it writes "Data pending" rather than guessing.

Why this matters to you. An SEO report you hand a client is only as good as your credibility. This system is designed so you never present a number that later turns out to be invented. When it says a section is "pending", that's a feature — it's protecting you.

Where the data comes from

Two kinds: on-site (the system reads the website's own pages and code — always available, no account needed) and off-site (rankings, keyword volumes, backlinks, Google Business Profile — these need an API account you connect during onboarding).

4. First launch & the /seo-setup wizard

Your very first task is connecting your data sources. There are two equivalent ways; pick whichever feels comfortable.

Option A — let Claude guide you (recommended)

1 Start Claude Code: type claude and press Enter.
2 Type /seo-setup and press Enter.
3 Claude explains each data provider, what it unlocks, and where to get the key, then walks you through connecting them. Follow along conversationally.

Option B — run the wizard directly in your terminal

Inside Claude Code you can run a terminal command by starting the line with !. Run:

! python ~/.claude/skills/seo/onboarding/setup_wizard.py

The wizard prints a menu, asks for each key, checks it against the live service, and stores it securely. Both options do the same thing.

Golden rule — never paste an API key into the chat. Keys are secrets. The wizard hides them as you type (you won't see the characters — that's normal). If you ever accidentally paste a key into the conversation, treat it as exposed and regenerate it from the provider's dashboard.

What "connecting a provider" actually does

  1. You paste the key into the wizard's hidden prompt (in your terminal, not the chat).
  2. The wizard validates it live — so a typo is caught immediately, not three audits later.
  3. It stores the key in a protected folder on your computer (see section 14).
  4. For data connectors, it wires up the connection so Claude can use it automatically.
After connecting anything, restart Claude Code so it loads the new connections. Then type /seo-setup verify to confirm everything is green.

5. Connecting your APIs (one by one)

Here is each provider, what it unlocks, and exactly where to get the credential. Connect the ones you need; skip the rest.

DataForSEO Most valuable

Unlocks: keyword rankings, the full keyword-research suite, competitor analysis, business listings, and AI-mention tracking — the engine behind most off-site data.

Get it: register at app.dataforseo.com/register. In the dashboard, find your API password (this is different from your login password) under API Access. You'll enter your login email + that API password.

The #1 gotcha — IP whitelist. DataForSEO can be set to only accept calls from approved IP addresses. If your data calls fail with error 40207, your computer's internet (IP) address isn't on the approved list. Fix it at app.dataforseo.com/api-access — either add the IP it shows you, or turn the whitelist off. If you're on home/mobile internet your IP changes often, so turning the whitelist off is usually simplest.

Google API key Recommended

Unlocks: real Core Web Vitals "field data" (how fast the site actually is for real visitors) and PageSpeed.

Get it: at console.cloud.google.com/apis/credentials, create an API key. Then enable two APIs on that key's project: "PageSpeed Insights API" and "Chrome UX Report API".

Google Search Console + GA4 Optional · skippable

Unlocks: indexation status, search performance (clicks, impressions, average position), and organic traffic from Analytics.

Get it: this one uses a sign-in flow (OAuth), not a simple key. In Google Cloud, create an "OAuth client → Desktop app" and download its client_secret.json. The wizard records it and gives you a one-time sign-in command to run. Make sure the Google account you sign in with has access to the Search Console property and the GA4 property.

Don't have access yet? Choose "attach later" in the wizard (section 6). The audit still runs — these sections just show "Data pending" until you connect them.

Google Business Profile Optional · owner only

Unlocks: first-party local insights — real impressions, calls, direction requests, and posts from the business's Google listing.

Get it: also a sign-in flow, requiring the "Google Business Profile API" enabled and the signed-in account to have owner/manager access to the business listing.

No owner access? No problem. Skip it. The local audit automatically falls back to DataForSEO's public Google-Business data, so you still get a full Local SEO section — just without the owner-only metrics.

Firecrawl & Exa Optional

Firecrawl (firecrawl.dev/app/api-keys) handles crawling very large or JavaScript-heavy sites. Exa (dashboard.exa.ai/api-keys) adds neural web search for competitor and entity research. Both are simple API keys.

6. "Attach later" & checking status

Connecting things when you're ready

You rarely have every credential on day one. The sign-in providers (Search Console/GA4 and Business Profile) let you choose, in the wizard:

ChoiceWhat happens
[N]owConnect it immediately, including the one-time sign-in.
[L]aterMark it "pending". The audit runs without it and clearly labels the missing sections. Connect it whenever you're ready.
[S]kipDon't use this provider at all for now.

Connecting (or replacing) just one provider

# add or re-do a single provider (e.g. dataforseo, exa, gbp)
! python ~/.claude/skills/seo/onboarding/setup_wizard.py --provider dataforseo

Checking what's connected

Any time, run a status check:

! python ~/.claude/skills/seo/onboarding/setup_wizard.py --check

You'll see each provider marked connected, pending (attach later), or not configured — plus whether your stored keys have safe file permissions. Inside Claude you can also just say /seo-setup verify and Claude reads this for you.

A "pending" provider is a reminder, not an error. The system is telling you exactly what's left to unlock — and the audit will keep working in the meantime.

8. The flagship: a full site audit

This is the command you'll use most. Typing /seo audit https://yourclient.com runs a four-phase pipeline. You don't drive it — but knowing the phases helps you read the result.

Phase 0 — Understand the business (first, always)

Before looking for problems, the system reads the site to figure out what the business actually is: its model (B2B, e-commerce, local, etc.), country of origin, likely target markets, ideal customers, and the keyword themes that matter. This shapes everything after it — there's no point researching keywords for the wrong country. You'll see this summarised at the top of the report as "Business Understanding".

Phase 1 — The audit itself

Specialist agents examine the site in parallel across: technical, content/E-E-A-T, schema, Core Web Vitals (real speed data), AI-search readiness, and search-experience. Each finding is checked against the evidence; high-severity findings are double-checked before they make the report.

Phase 2 — Keyword research

Using the business understanding, it runs a full keyword-research pass across the home market and detected target markets (section 9).

Phase 3 — Local SEO & Google Business Profile

If the business is local (has premises or a service area), it audits the Google Business Profile and local signals (section 10). If it's purely online, this phase is skipped and the report says so.

What to expect when you run it

1 You type /seo audit https://yourclient.com.
2 The system fetches the site's pages, builds the business profile, and fans out its specialists. This takes a few minutes — longer for big sites.
3 You get a written summary in the chat, and full report files saved to disk (an action plan, a keyword report, and — on request — a polished PDF).
Tip: if it's a big site, the system honours sensible crawl limits and tells you what it covered. If something couldn't be fetched, it says so rather than guessing.
Cost awareness. The keyword and local phases use DataForSEO credits (typically well under a few dollars for one audit). The system can show a cost plan before spending — just ask it to preview costs if you want to see them first.

9. Keyword research, explained

The keyword research is built to be decision-ready, not a raw dump. It researches your seed themes across the home market plus any target markets the system detected, and returns a table with the numbers you need to prioritise.

The metrics you'll see

ColumnWhat it tells you
Global / Country volumeHow many people search this per month (worldwide, and per market)
CPCWhat advertisers pay per click — a proxy for commercial value
KD (difficulty)0–100; how hard it is to rank. Lower is easier.
IntentWhy people search it: informational, commercial, transactional, navigational
Current rankWhere the site already ranks (blank = not yet ranking)

Opportunity tiers — where to act first

  • Quick wins — keywords you already rank for at positions 4–20. Small on-page work moves these onto/up page one. Highest ROI; start here.
  • Strategic targets — high-volume commercial terms you don't yet rank for → new or upgraded pages.
  • Gap captures — terms a competitor ranks for and you don't.
  • Long-tail / content — lower-volume, specific terms; great for blog posts and AI-citation answers.
Reading the markets honestly. Sometimes a "target market" turns out to have very little search demand. The report will say so plainly and tell you to focus where the volume actually is, rather than spreading effort thin.
If you see "Data pending — requires DataForSEO". Keyword research needs DataForSEO connected and (if a whitelist is on) your IP approved. See sections 5 and 16.

10. Local SEO & Google Business Profile

For any business with a physical location or service area, this phase audits how it shows up locally. It works on three levels depending on what you've connected:

TierNeedsWhat you get
A — First-partyBusiness Profile connected (owner access)Real impressions, calls, direction requests, posts — the deepest view
B — PublicDataForSEO connectedMap-pack rankings, GBP completeness, reviews & velocity, competitors in radius
0 — On-page onlyNothing extraLocal signals on the website itself (NAP, local schema, location pages)

The report always states which tier it used, so you know what's covered and what would deepen it.

What it checks

  • NAP consistency — is the Name, Address, Phone identical across the site, Google, and directories? Mismatches hurt local ranking.
  • Reviews — rating, count, how steadily new reviews arrive (velocity), and owner responses.
  • Profile completeness — categories, hours, photos, attributes.
  • Local schema — is the website marked up as a LocalBusiness with location and hours?
Real example from a live audit: a client's address had a stray trailing dash ("Dubai -") in both the website code and the Google profile — a single bad master record polluting everything. The local audit catches exactly this kind of cross-source inconsistency.

12. Reading your reports

Reports are written to files on your computer (Markdown for editing, and a polished PDF on request). Here's how to read them with confidence.

The health score

A 0–100 composite. Importantly, the system only scores what it can verify. If backlinks aren't connected, that category shows "Pending" and is excluded from the score — and the report says so. A score is never padded with guesses.

Severity labels

LabelMeaningAct within
CriticalBlocks indexing or causes penaltiesImmediately
HighSignificantly affects rankings~1 week
MediumWorthwhile optimisation~1 month
LowNice to haveBacklog

Every finding shows its evidence

Findings cite where they came from — a file and line in the site's code, or a specific data field. This is what lets you (or a developer) verify and fix each item, and what lets you defend the report to a client.

"Data pending" sections

These aren't failures — they're an honest statement that a data source isn't connected yet. The report tells you exactly what to connect to fill them in.

The action plan

Findings are turned into a prioritised plan: each item has an effort estimate and the reason it matters, ordered so dependencies come first (e.g. fix the server speed before chasing layout-shift ghosts). Work it top-down.

13. A suggested weekly routine

A simple rhythm that turns the system into a repeatable service:

1 Monday — full audit per client. /seo audit on each site. Skim the score and the top-5 priorities.
2 Act on quick wins. From the keyword report, take the "quick win" keywords (already ranking 4–20) and improve the matching pages first — best effort-to-result ratio.
3 Track change. Keep a /seo drift baseline per site; compare weekly so client updates are concrete.
4 Re-measure at ~28 days. Rankings and Core Web Vitals are trailing windows — wait about a month after changes to fairly judge impact, then re-audit.
For a new prospect, run one audit live on the discovery call. A real score and three evidenced priorities win engagements faster than a promise to "take a look".

14. Where your keys live (security)

You're handling client and account credentials, so it's worth knowing exactly how they're treated.

  • Stored on your machine only, in a protected folder: ~/.config/claude-seo/ (on Windows, under your user profile). Keys are saved with owner-only permissions.
  • Never in the project files, never uploaded anywhere. The only places a key travels are to that provider's own service when the system calls it.
  • Never shown in chat. The wizard hides key entry; the system is instructed never to print a key back.
  • No tracking. The system doesn't phone home or send analytics.

Good habits

  • If you ever paste a key into the chat by mistake, regenerate it from the provider.
  • To remove a provider, delete its file in ~/.config/claude-seo/.
  • Run /seo-setup verify occasionally — it flags if any stored key has loose file permissions.
In short: your credentials stay on your computer, owner-only, and are used solely to call the services you connected. This is by design and is documented in the project's SECURITY.md.

15. Keeping the system up to date

Claude SEO Pro is built on an open-source SEO toolkit that improves over time. The Pro layer (your onboarding, the smarter audit pipeline, the no-fabrication rules) sits on top and is preserved across updates.

You don't need to do anything manually — if your team has enabled it, updates arrive as a reviewed pull request roughly weekly, pulling in upstream improvements while keeping all the Pro features intact. Whoever maintains the repository approves these.

As a day-to-day user, the practical takeaway is: the system stays current on its own, and updates never silently change your connected APIs or your reports' integrity rules.

16. Troubleshooting

SymptomCause & fix
Data calls fail with 40207DataForSEO IP whitelist. Your computer's IP isn't approved. Add it (or turn the whitelist off) at app.dataforseo.com/api-access. On home/mobile internet your IP changes, so turning the whitelist off is usually best. To see your current IP, ask Claude or run ! curl -s https://api.ipify.org.
Backlinks section says 40204 / "subscription required"Your DataForSEO plan doesn't include the Backlinks API. Activate it at app.dataforseo.com/backlinks-subscription, or connect free Moz / Bing Webmaster credentials. The audit deliberately leaves this empty rather than estimate it.
A data command says the connector "isn't available"Either the provider isn't connected (run /seo-setup verify) or Claude Code wasn't restarted after connecting it. Restart Claude Code — connectors load at startup.
DataForSEO / Firecrawl / Exa won't load at allNode.js missing. These connectors need it. Install Node from nodejs.org, then restart.
Keyword research shows nothing for a marketSome markets/languages aren't supported by the keyword-ideas engine (you'll see a note), or the market genuinely has low demand. The report uses the best available data and says which.
Google Search Console / GBP sign-in didn't finishThe one-time browser sign-in needs to complete. Re-run the auth command the wizard gave you, and make sure the Google account has access to the property/listing.
A report section says "Data pending"Working as intended — that data source isn't connected. The section names exactly what to connect.
An audit couldn't read some pagesThe site may block automated requests, or pages may be down. The report lists what it couldn't fetch and never fills the gap with guesses.

17. FAQ

Do I need to know how to code?

No. You type commands and read reports. The only "commands" are the copy-paste lines in this manual.

Will it ever make up data to fill a report?

No — that's its core design rule. Missing data shows as "Data pending" with instructions, never as an invented number.

How much do the APIs cost?

The on-site audit is free. DataForSEO charges small per-call amounts; a full audit's keyword + local research is typically well under a few dollars. You can preview costs before running.

Can I run it on any website?

Yes — you don't need to own the site for on-site analysis, rankings, keyword research, or public local data. Only the first-party Google Search Console / GA4 / Business Profile data requires access to that account.

What's the difference between /seo local and /seo maps?

/seo local checks local signals on the website; /seo maps checks the business on map platforms (Google Maps rankings, GBP, reviews). The full audit uses both where relevant.

How often should I re-audit?

Weekly for active clients is ideal; always wait ~28 days after changes before judging ranking/speed impact, since those metrics are trailing.
